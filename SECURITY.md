# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in this project, please report it privately. **Do not open a public GitHub issue.**

Email: **hakan@ksenda.com**

Please include:
- A description of the vulnerability
- Steps to reproduce
- Affected versions or commits
- Potential impact

You should receive an initial response within 72 hours. We will work with you to understand and remediate the issue, and will credit you in the fix announcement unless you prefer otherwise.

## Scope

This policy covers the code in this repository. For vulnerabilities in third-party dependencies, please report them upstream and let us know so we can update.

## Supported Versions

Only the latest commit on the default branch is actively maintained. Older tags and branches do not receive security updates.

## Out of Scope

- Reports generated by automated scanners without a concrete proof-of-concept
- Social engineering attacks against maintainers or users
- Issues in third-party services we integrate with (please report to those vendors)
