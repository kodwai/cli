## How to Contribute

### Reporting Bugs

1. Search [existing issues](../../issues) first to avoid duplicates.
2. If you find none, open a new issue using the **Bug Report** template.
3. Include reproduction steps, expected vs. actual behavior, and environment details.

### Suggesting Features

Use the **Feature Request** template. Explain the use case, not just the feature.

### Pull Requests

1. Open an issue first if the change is substantial (more than ~50 lines or touches public API).
2. Fork the repo, branch from `main`.
3. Make focused commits with clear messages.
4. Run tests and linters locally before pushing.
5. Open a PR referencing the related issue.
6. Be patient — reviews happen when maintainers have bandwidth.

### Commit Messages

Keep them concise and present-tense:
- `Add X to Y` (good)
- `Fix race condition in Z` (good)
- `updated stuff` (avoid)

## Code Style

Follow the conventions already used in the codebase. If a linter or formatter is configured, run it.
